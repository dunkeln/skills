# Gotchas

- Fake parallelism: four lanes repeat one hypothesis with different labels.
- Premature implementation: code changes begin before the winner survives opposition.
- Confidence laundering: the most fluent lane is treated as the strongest lane.
- Generic adversary: critique is broad caution rather than a concrete disproof attempt.
- Verification theater: checks prove the code runs but not that the chosen lane was right.
- Scope creep: the Explorer lane turns uncertainty into a refactor.
- Safety theater: the Conservative lane avoids the core issue and preserves bad behavior.
- Memory blindness: the skill ignores AGENTS files, local memories, prior fixes, or analogous flows.
- Tool confusion: an incident RCA should use `postmortem`; a future go/no-go should use `premortem`.
- Slowdown on obvious fixes: 4x4 is for ambiguity, not every edit.
